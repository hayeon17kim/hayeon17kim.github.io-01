메서드에 선언되는 변수는 로컬 변수고, 이것은 스택에 만들어짐





인스턴스에 메서드가 있는 게 아니라 변수가 있는 것

```java
package com.eomcs.util;

public class Calculator {
  // result 변수는 각 계산마다 개별적으로 관리해야 한다.
  // => non-static 변수로 선언하라!
  // => non-static 변수는 new 연산자를 통해서만 만들 수 있다.
  // => non-static 변수는 Heap 영역에 생성된다. 
  public int result = 0;
  
  // static 매서드는 non-static 변수를 사용할 수 없다.
  // 사용하려면 new 명령을 통해 생성된 메모리의 주소를 알아야 한다.
  // 
  
  // non-static 메서드(인스턴스 메서드)는
  // 인스턴스 주소를 받는 this 라는 빌트인 변 수가 있다.
  public void plus(int value) {// 로컬변수는 메서드가 호출될 때 stack 에 만들어짐
    this.result += value;
  }
    
  public void minus(int value) {
    this.result -= value;
  }
    
  public void multiple(int value) {
    this.result *= value;
  }
    
  public void divide(int value) {
    this.result /= value;
  }
}

```

엄밀히 말하면 calculator는 데이터 타입이 아님. result변수가 있는 것은 사실이지만,, 데이터라고 한다면 member, project, 같은거. 클래스의 대부분은 메소드 (기능)을 묶은 클래스. 그 중 5~6 프로가 데이터.

Date 클래스, String 클래스 

System 클래스, FileInputStream클래스 등등 기능을 묶은 클래스

기능을 묶은 클래스도 인스턴스 필드가 있다는 거. 

테스크 클래스의 변수들하고 오늘 사용한 거하고 calculator의 result와는 변수의 목적이 다름.
