# 소프트웨어 생명주기

### 폭포수 모델

- 이전 단계로 돌아갈 수 없는 모델
  - 각 단계를 확실히 매듭지어야 함
  - 2개 이상의 과정을 병행할 수 없음
  - 매뉴얼 작성 필요
  - 개발 이후 발생된 오류 수정 불가
- 타당성 검토 => 계획 => 요구 분석 => 설계 => 구현 => 검사 => 유지보수



### 프로토타입

- 인터페이스 중심 개발 
- 폭포수 모형의 단점(개발 이후에 발생하는 오류 해결 못하는 점) 보완
